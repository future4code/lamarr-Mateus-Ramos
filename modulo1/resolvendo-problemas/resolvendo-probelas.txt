
Durante a aula, nos foi apresentado 7 passos essencias para solução de problemas. O primeiro dele diz respeito a atenção que a gente deve ao enunciado. Este ponto é extremamente importante por que é o nosso primeiro contato com o problema a ser resolvido logo, entendê-lo é essencial para desenvolver todos os outros passos. É possível fazer uma analogia da importância desse ponto com outras áreas, nesse sentido, posso citar a a resolução de problemas físicos e matemáticos onde, normalmente, o enunciado traz informações extremamente importantes para a resolução de todo o problema, então entender ele é um passo fundamental para o êxito do processo.
Retira informações relevantes, que é o segundo passo essencial apresentado, está diretamente ligado ao primeiro. Uma vez que você entende o enunciado, você consegue quase que de forma estantânea tirar as informações que vão permitir o sucesso na resolução do problema, neste caso, analisar as condições, as possíveis variáveis, possibilidade de criação de objetos entre outras.
O terceito passo, pensar entradas e saídas ajudam a entender o que esperamos do nosso código, o que nos leva a refletir sobre uma solução para otimizar as saídas tornando nosso código mais eficiente. O esquema visual, 4 passo essencial, ajuda a entender toda estrutura do código, facilita a visualização de entradas, saídas e funções necessárias para solucionar o problema. Criar codigo feio é um ponto interessante de ser discutido especialmente para quem está aprendendo. O código feio permite que a gente tente criar a solução sem se preocupara tanto com a estruturação do código, após isso, a gente ponde melhorar o código colocando a estrutura necessária.
Testar na força bruta auxilia para teste. A gente consegue testar a solução do problema sem aplicação de alta performance, podemos usar para criar para entender nosso objetivo antes de deixar nosso código completo.
Por fim, o refatorar seria o equivalente a otimização do código. Buscar uma forma de deixa-lo eficiente e otimizado.
